1. Resumen Ejecutivo

Se desarrollo un sitio web institucional completo para el DIF San Mateo Atenco, que incluye un sistema de gestion de contenido (CMS) personalizado que permite al personal del DIF administrar todo el contenido del sitio sin necesidad de conocimientos tecnicos.

Alcance del Proyecto

ComponenteDescripcion
Sitio PublicoPagina web institucional con multiples secciones, responsiva para web, tablet y movil
Panel de AdministracionCMS completo con 25+ secciones administrables
Base de Datos30+ tablas MySQL con relaciones, indices y cascadas
Sistema de UsuariosRoles (admin/usuario) con permisos granulares por seccion
Transparencia DinamicaMotor de plantillas para crear secciones de transparencia ilimitadas

5.2 Funcionalidades Transversales

  • Multi-upload: Subir multiples imagenes simultaneamente (Slider Comunica, Galeria, Voluntariado)
  • Drag & Drop: Reordenar imagenes arrastrando tarjetas (SortableJS con AJAX)
  • Flash Messages: Notificaciones de exito/error despues de cada accion
  • Modales de Confirmacion: Confirmacion antes de eliminar registros
  • Upload Handler: Validacion de tipo, tamano (20MB max), renombrado seguro con hash MD5

6. Sistema de Usuarios y Permisos

6.1 Roles

RolDescripcionAcceso
adminAdministrador unico del sistemaAcceso total a todas las secciones + gestion de usuarios
usuarioUsuario publico creado por el adminSolo las secciones asignadas por el administrador

6.2 Funcionalidades

  • Crear usuarios con nombre, usuario y contrasena
  • Asignar permisos granulares por seccion (checkboxes)
  • Activar/desactivar usuarios sin eliminarlos
  • Resetear contrasena de cualquier usuario
  • Eliminar usuarios (cascada elimina permisos)
  • El sidebar y dashboard se filtran automaticamente segun permisos
  • El auth_guard bloquea acceso a secciones no permitidas

6.3 Flujo de Autenticacion

PasoDescripcion
1Usuario ingresa credenciales en login.php
2Se verifica rate limiting (max 5 intentos en 15 min)
3Se consulta la tabla admin con prepared statement
4Se verifica password con password_verify() (bcrypt)
5Se verifica que el usuario este activo
6Se regenera session ID y se guardan rol, id, username en sesion
7auth_guard.php verifica permisos en cada pagina
8Sesion expira automaticamente a los 5 minutos de inactividad

7. Transparencia Dinamica

Sistema que permite crear secciones de transparencia ilimitadas eligiendo una plantilla predefinida.

7.1 Plantillas Disponibles

PlantillaEstructuraEjemplo Original
SEACBloques (anio) → Conceptos → PDFs trimestrales (Q1-Q4)SEAC
CONACBloques (anio) → Conceptos → PDFs trimestrales (Q1-Q4)CONAC
Cuenta PublicaBloques (anio) → Conceptos con PDF directoCuenta Publica
FinancieroBloques (anio) → Conceptos con PDF directoFinanciero
Presupuesto AnualBloques (anio) → Conceptos → Sub-anios con PDFPresupuesto Anual
PAETitulos globales → PDFs por anioPAE
MatricesAnios con PDF directo (estructura simple)Matrices de Indicadores

7.2 Funcionamiento

  • El admin crea una seccion con nombre y plantilla desde transparencia_dinamica.php
  • Se genera un slug unico automaticamente
  • El contenido se gestiona desde transparencia_seccion.php con interfaz adaptada a la plantilla
  • Las secciones activas aparecen automaticamente en el menu de Transparencia del sitio publico
  • El front renderiza la seccion en seccion_dinamica.php con el estilo exacto de la plantilla original

8. Seguridad

MedidaImplementacion
ContrasenasHash bcrypt via password_hash() / password_verify()
SQL InjectionPrepared statements PDO en todas las consultas
XSShtmlspecialchars() en toda salida de datos
CSRFTokens CSRF en todos los formularios POST
Rate LimitingMax 5 intentos de login en 15 minutos por IP
Session TimeoutExpiracion automatica a los 5 minutos de inactividad
Session Fixationsession_regenerate_id(true) al iniciar sesion
Upload ValidationValidacion de tipo MIME, extension, tamano (20MB max), renombrado MD5
PDF Anti-CopyRenderizado via canvas (PDF.js), bloqueo de Ctrl+P/S/C, PrintScreen
Directory TraversalRutas de upload controladas con BASE_PATH
